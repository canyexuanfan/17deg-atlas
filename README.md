# 17deg Atlas

让 AI Agent 帮你创建、连接和使用自己的知识库——技术预览版 0.1.0

## 项目价值

17deg Atlas 是一个开源的个人知识管理工具，让你通过自然语言与 AI Agent 协作，构建和使用属于你自己的知识库。本地运行的 Agent 与远端托管的 Agent 使用各自独立入口，在你确认的范围内保存、检索和核对知识来源。

**核心理念：**
- 你的知识，你的控制。内容是否公开、分享给谁、如何授权，始终由你决定
- 支持 AI Agent 参与知识库的构建、维护和智能检索，让知识管理更高效
- 区分本地和远端使用场景，提供符合安全预期的独立入口

## 适合谁？

- **个人知识管理者**：希望用 AI 辅助整理、检索和复用个人知识
- **Agent 用户**：希望让自己的 Agent 能够访问真实、可验证的知识来源
- **开源爱好者**：寻找可自托管、可扩展的知识管理解决方案

## 核心能力

### 本地 Agent 入口（`skills/17deg-atlas-local`）
- **创建与连接**：在本地设备创建新知识库，或连接现有知识库
- **灵活存储**：支持明文或 age 加密存储（五级分类），满足不同安全需求
- **授权检索**：Agent 可检索你明确授权的知识内容
- **来源摄入**：支持从多种来源导入知识（网页、文件、对话记录等）
- **知识编译**：将原始信息整理为结构化、可复用的知识条目
- **语义审核**：协助检查内容质量和一致性
- **能力 Skill 编译**：将验证过的知识转化为可复用的 Agent 能力
- **锁定与恢复**：支持知识库锁定和完整性验证

### 远端 Agent 入口（`skills/17deg-atlas-remote`）
- **公开读取**：Agent 可读取你标记为公开的知识
- **加密追加**：支持向加密知识库安全追加新内容（默认不解密已有内容）
- **受控授权检索**：仅检索你明确授权的知识范围
- **连接检查**：验证远端 Agent 与知识库的连接状态

## 快速开始

### 选择适合的入口

根据你的 Agent 运行位置，选择对应的入口 Skill：

**重要：** 你在每个环境只需要安装一个入口，工具不会猜测或自动切换。

### 本地 Agent 完整提示词

复制以下完整提示词给你的本地 Agent：

```
请安装并使用这个 Skill：https://github.com/canyexuanfan/17deg-atlas/tree/main/skills/17deg-atlas-local 。如果不能直接安装 GitHub 子目录，请自动克隆 https://github.com/canyexuanfan/17deg-atlas 并加载 `skills/17deg-atlas-local`，不要让我手工执行命令。默认把入口 Skill 安装在当前项目内；除非我明确要求，不要安装为全局 Skill。安装后，在当前设备为我创建或连接知识库；需要 GitHub 操作时先向我确认，不要自行改成纯本地流程，并一直执行到工具确认首次连接完成。完成后直接向我报告，不要自行追加其他验收或安装。
```

### 远端 Agent 完整提示词

复制以下完整提示词给你的远端 Agent：

```
请安装并使用这个 Skill：https://github.com/canyexuanfan/17deg-atlas/tree/main/skills/17deg-atlas-remote 。如果不能直接安装 GitHub 子目录，请自动克隆 https://github.com/canyexuanfan/17deg-atlas 并加载 `skills/17deg-atlas-remote`，不要让我手工执行命令。优先安装在当前项目；如果平台只支持固定 Skill 目录，则使用平台标准目录，不额外创建其他副本。安装后，把当前远端 Agent 连接到我的知识库，并一直执行到工具确认首次连接完成。完成后直接向我报告，不要自行追加其他验收或安装。
```

Skill 会自动准备项目所需工具，普通用户无需安装 CLI 或手工输入命令。

## 首次运行会确认什么？

安装过程中，Agent 会在执行以下操作前向你确认：

- **联网安装**：从 GitHub 下载项目文件
- **GitHub 登录**：使用 GitHub CLI 进行身份验证
- **创建仓库**：在 GitHub 上新建知识仓库
- **连接现有仓库**：访问你已有的 GitHub 仓库
- **首次同步**：初始化知识库内容
- **仓库公开性**：设置仓库为公开或私有
- **加密凭据**：测试模式可创建仅供验证的测试凭据；长期使用的凭据不会在未确认时自动生成
- **权限设置**：配置 Agent 的访问权限
- **删除操作**：删除任何知识库内容或仓库

其他操作（如检测 GitHub CLI 或 age 是否已安装）会自动完成。

## 日常使用示例

安装完成后，你可以直接用自然语言与 Agent 协作：

### 知识保存与整理
- “保存这段内容到我的知识库。”
- “把这篇文档整理成结构化知识条目。”
- “将刚才的对话记录保存为可复用的知识。”

### 知识检索与使用
- “搜索我关于这个主题的所有资料。”
- “我之前存过相关内容，帮我找出来。”
- “根据我的知识库，总结这个问题的要点。”

### 内容发布与分享
- “把这篇知识整理成可公开的版本。”
- “制作一个分享给团队的版本，隐藏敏感信息。”
- “生成这篇知识的 Markdown 发布格式。”

### 能力复用
- “把刚才那个已验证的方法做成可复用能力，安装到项目位置。”
- “将这个工作流转化为 Agent 技能，供以后使用。”

能力安装位置由你选择：仅在当前项目、全局可用，或让 Agent 根据运行时场景推荐。首次全局安装会征求你的确认。

## 开发者快速运行

- Python >= 3.10

```bash
python scripts/17deg-atlas.py --help
```

```bash
python -m unittest discover -s modules/knowledge/tests -v
```

## 当前边界

0.1.0 是技术预览版，当前重点是个人使用场景中的本地知识管理与 GitHub 连接。不同 Agent 对 Skill 安装和工具调用的支持程度可能不同；入口会在真实联网、仓库、公开、密钥、权限或删除动作前暂停确认。请先使用测试模式验证流程，不要把测试凭据当作长期凭据。
