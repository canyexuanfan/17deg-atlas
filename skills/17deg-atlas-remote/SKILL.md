# 17deg Atlas Remote

仅在明确选择的远端环境中执行。环境不匹配时停止并提示使用本地入口；不得猜测或自动切换角色。

## 首次连接

优先安装在当前项目；平台只支持固定 Skill 目录时使用平台标准目录，不额外创建其他副本。

1. 必要时运行 `scripts/bootstrap.py`。
2. 通过 `scripts/atlas.py workspace plan` 生成计划。
3. 将同一计划中需要用户决定的真实动作汇总为一份确认清单；平台要求逐项确认时遵循平台要求。
4. 通过 `scripts/atlas.py workspace start` 执行相同计划。
5. 当结果同时包含 `onboarding_complete: true` 和 `terminal_state: complete` 时，向用户报告成功并停止。

工具返回多个连接或新建选项时，逐项标注现有内容去留与目标位置，等待用户选择；用户未明确选择前禁止新建、连接或迁移，保持现有内容不变。

若旧实例选项中的 `migrate-current` 标记 `execution_entry=local`，远端 Agent 不得复制 identity、不得降低验证强度、也不得自行模拟迁移流程；应保持旧实例原样不动，并提示用户在受控本地环境中安装本地入口以完成迁移。远端可在用户明确选择后继续沿用旧结构或新建空实例，但必须明确告知：新建空实例不会复制旧实例的任何内容。

完成后不得自行追加再次连接、额外安装或其他验收动作。

## 自然语言路由

用户只描述目标。Agent 只调用当前受信运行时明确授权且已部署的能力，生成参数并统一调用 `scripts/atlas.py remote`：

- 连接或公开读取：使用对应的连接与读取子命令。
- 保存资料：按目标安全级别追加远端事件，不在远端静默公开。
- 授权检索：仅在受信运行时和明确授权均满足时执行，结束后清除临时结果。

普通无解密远端不编译私密能力，不提升权限或自动换入口。默认没有解密能力，也不声称完成本地 KB2 的完整知识整理；远端写入的内容可由有权限的本地流程继续编译。

远端可将运行结果和反馈安全追加为事件。taxonomy 审核、关系晋升、适用范围投影和重编译仍交由有权限的本地/受控流程处理；不得在远端静默形成 canonical。

## 确认边界

网络安装、GitHub 真实写入、扩大访问范围、公开和删除必须在执行前获得明确确认。

Skill 只负责编排 CLI。权限校验和写入合同以 CLI 返回结果为准，不在 Skill 内复制实现。
