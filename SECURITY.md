# 安全政策

本政策适用于 17deg-atlas 仓库的 main 分支。

## 漏洞报告

**优先方式：请使用 GitHub Private vulnerability reporting 私下报告安全问题。**

如该入口不可用，请创建普通 Issue，标题注明 "[安全]" 字样，**不要在 Issue 中包含任何漏洞细节**，仅请求安全联系渠道。

## 报告内容

请在报告中包含：
- 受影响的版本号
- 复现条件与最小复现步骤（仅在私下报告中提供）
- 潜在影响评估
- 修复建议（如有）

**禁止公开：**
- 密钥、凭据或敏感配置
- 个人知识库内容
- 可被直接利用的漏洞细节

## 处理流程

1. 维护者收到报告后将进行确认
2. 确认后协调修复与披露时间表
3. 修复完成后发布安全更新

**不承诺具体响应时限。**

## 依赖漏洞

如发现以下依赖的安全问题，请直接向其上游报告：
- age 加密工具
- GitHub API
- 其他第三方依赖

感谢帮助提升 17deg-atlas 的安全性。
